Projet SAE 302 — Gestion des Sauveteurs Spéléologues Application de gestion des équipes de secours du Spéléo-Secours Français AUTEURS Pape Birame Fall Sokhna Gueye Perceval Behanzin Mohamed Fall PROFESSEURS RESPONSABLES Benoit Tremblais Vincent Verdon (client) OBJECTIF Créer une application graphique permettant de gérer les sauveteurs spéléologues, leur planning, leurs missions et leur disponibilité, avec synchronisation peer-to-peer sur un réseau local, on enverra seulement les données nécessaires pour les sauveteurs pas toute la base de données. APPROCHE: Pour les accès à l'application, tous les utilisateurs peuvent voir le planning. Seuls les gestionnaires et les administrateurs doivent s'authentifier pour faire leurs tâches selon leurs profils. On a choisis ce systeme car on a s'est mis d'accord que le planning doit être visible par tous les utilisateurs et ça ne represente pas un point d'entrée pour les hackers. Pour la base de données, on a opté pour SQLite car c'est plus facile à mettre en place et plus proche de MySQL qu'on a deja utilisé. Cette base sera locale à chaque poste et on fera la synchronisation entre les postes via des sockets TCP. Pour l'envoi des données, on a choisi le format JSON car c'est un format léger et facile à manipuler en Python. On enverra seulement les données nécessaires pour les sauveteurs modifiés, ajoutés ou supprimés. Un utilisateur ouvre l'application et voit le planning sans s'authentifier. 2. Un gestionnaire s'authentifie: - Ajoute un nouveau sauveteur avec ses spécialités. - Modifie le statut d'un sauveteur (disponible, en mission, indisponible). - Supprime un sauveteur. - Modifie le planning (horaires). 3. Un administrateur s'authentifie: - Gère les comptes utilisateurs (ajout, suppression, modification). - Voir le planning complet. 4. Lorsqu'un sauveteur est ajouté/modifié/supprimé, les autres postes reçoivent automatiquement la mise à jour via TCP.
